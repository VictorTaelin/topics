Introduction

Problem we solve:

function f: [i] -> [o]

it has linear inputs and ouputs; (this is a very common case because the user interacts linearly.)

Incremental modifications of input should incur incremental re-computations of the output.

Additional hypothesis: * the user observes a small portion of the output at a time * (the function is not cahotic (e.g. not sort))
